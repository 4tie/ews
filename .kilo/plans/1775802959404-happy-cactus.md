# Plan: Export Strategy Backtest to Per-Strategy Directory

## Goal
The current implementation already stores backtest results at `user_data/backtest_results/<strategy>/<run_id>.backtest.zip`. This plan verifies the implementation and fixes a related bug.

## Current State (Verified)
- `strategy_results_dir(strategy)` in `paths.py` returns `user_data/backtest_results/<strategy>`
- `_backtest_artifact_paths()` in `freqtrade_cli_service.py` uses this function correctly
- The export command already builds: `--export trades --export-filename user_data\backtest_results\<strategy>\<run_id>.backtest.zip`

## Bug Found
**Location:** `app/services/freqtrade_cli_service.py` line 76  
**Issue:** `dry_run_wallet` is passed to `build_backtest_command()` but the function doesn't accept this parameter.

```python
# Current (broken):
cmd = build_backtest_command(
    ...
    dry_run_wallet=payload.get("dry_run_wallet"),  # ERROR: no such parameter
    ...
)
```
