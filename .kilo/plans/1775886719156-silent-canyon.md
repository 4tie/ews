# Plan: Remove --backtest-directory from Backtest Command

## Problem
When running backtest from the UI app, the command includes `--backtest-directory` which causes connection issues:
```
--backtest-directory T:\Optimizer\user_data\backtest_results\MultiMa
```

The user wants the command to work WITHOUT the `--backtest-directory` flag.

## Root Cause
In `app/services/freqtrade_cli_service.py`, line 238, the `prepare_backtest_run` method passes `backtest_directory=artifacts["raw_result_dir"]` to `build_backtest_command`, which adds `--backtest-directory` to the command.

## Solution
Remove the `backtest_directory` parameter from the `build_backtest_command` call in `prepare_backtest_run` method.

## Files to Modify
1. `app/services/freqtrade_cli_service.py` - Line 238

## Changes
In `prepare_backtest_run` method, remove `backtest_directory=artifacts["raw_result_dir"]` from the `build_backtest_command` call.

The `artifacts` dict will still be returned (containing `log_file`, `raw_result_dir`, etc.), but the `--backtest-directory` flag won't be added to the command.
