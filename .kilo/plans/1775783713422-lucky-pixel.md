# Plan: Improved Time Range and Pairs Input

## Goal
Add better and easier time range and pairs selections to the backtesting setup panel.

## Changes Required

### 1. Date Input Improvements
**Files:** 
- `web/templates/pages/backtesting/backtesting.html` (lines 67-72)
- `web/static/js/pages/backtesting/setup/time-panel.js`
- `web/static/css/pages/backtesting.css` (likely needs new styles)

**Current:** Uses HTML5 `<input type="date">` which shows yyyy-mm-dd format

**Desired:** Use text inputs with mm/dd/yyyy placeholder and format, with date picker popup

**Implementation approach:**
- Replace with text inputs having mm/dd/yyyy placeholder
- Add a simple date picker or use a library like flatpickr
- Convert to yyyy-mm-dd internally for the backend
- Optional: use browser native date picker on mobile

### 2. Pairs Input Improvements  
**Files:**
- `web/templates/pages/backtesting/backtesting.html` (lines 78-87)
- `web/static/js/components/pair-input.js`

**Current:** Accepts single pairs or JSON paste, shows tags

**Desired:** Make it clearer how to input pairs - single pairs OR JSON

**Implementation approach:**
- Improve placeholder text
- Add better feedback when JSON is parsed
- Support `pair_whitelist` key in JSON
- Also support array format like `["BTC/USDT", "ETH/USDT"]`

### 3. Time Panel JavaScript Updates
**Files:**
- `web/static/js/pages/backtesting/setup/time-panel.js`

**Current:** Saves dates as yyyy-mm-dd

**Needed:** Parse mm/dd/yyyy input and convert to yyyy-mm-dd for storage

## Risk Assessment
- **Low risk** - UI only changes
- Existing state format already stores dates - just need to handle conversion
- No backend changes needed

## Out of Scope (for this iteration)
- Adding presets for common date ranges (This Week, Last Month, etc.)
- Timezone handling improvements
- Additional pair validation against exchange
